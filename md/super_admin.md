## 🛠️ Automated Setup Process

The system is designed to self-configure during the server startup (`npm run dev` or `npm start`). The seeding logic performs the following actions:

1.  **Role Verification**: Checks for the existence of the `SUPER_ADMIN` role in the `all_roles` table. Creates it if missing.
2.  **Department Verification**: Checks for the `ADMINISTRATION` department. Creates it if missing.
3.  **User Verification**: Checks if a user with the email `superadmin@kajlagbe.com` exists.
4.  **Seeding**: If the user is missing, it creates the Super Admin account, hashes the password, and links it to the appropriate role, department, and profile.
